Wat zijn datatypes?
Het juiste formaat voor elk attribuut
▼
🔙 Even terugkijken — Module 1
In module 1 leerde je dat een entiteit een object is (bijv. employee) en dat attributen de eigenschappen zijn (bijv. firstname, salary). Maar hoe weet de database wát voor soort waarde erin mag? Dat regelt het datatype.

Wat is een datatype?

Een datatype vertelt de database wat voor soort informatie een attribuut mag bevatten. Zonder datatype zou je een leeftijd van "appel" kunnen invullen — en dat wil je niet. 🍎

Klik de kaartjes om de drie redenen te ontdekken waarom datatypes belangrijk zijn:

✅ Validatie klik om te onthullen
Een attribuut accepteert alleen waarden die bij het datatype passen. Geen letters in een getal-veld, geen ongeldige datums.
⚡ Efficiëntie klik om te onthullen
Het juiste datatype gebruikt precies genoeg geheugen. Een INT voor een leeftijd kost minder ruimte dan een TEXT.
🔒 Data-integriteit klik om te onthullen
Datatypes begrenzen lengtes en formaten. Een landcode is altijd precies 2 tekens — nooit meer, nooit minder.
🎵 Real-world voorbeeld
Spotify slaat een song op. Hoe lang is de songtitel? Variabel → VARCHAR. Hoeveel streams? Een heel getal → INT. Wanneer uitgebracht? Een datum → DATE.
Elk attribuut heeft zijn eigen datatype — precies goed afgesteld.

Tekst datatypes
CHAR, VARCHAR en TEXT
▼

Voor het opslaan van tekst zijn er drie hoofdtypes. Ze lijken op elkaar, maar het verschil maakt veel uit voor de performance van je database.

📌 CHAR(n) — Vaste lengte

Altijd precies n tekens, ook als de waarde korter is. Lege posities worden opgevuld met spaties.

Gebruik voor: waarden die altijd even lang zijn.

countrycode CHAR(2)
-- 'NL', 'DE', 'FR'
-- altijd 2 tekens
productcode CHAR(5)
-- 'P1234', 'A0042'
-- altijd 5 tekens
📌 VARCHAR(n) — Variabele lengte

Tot maximaal n tekens, maar slaat alleen de werkelijke lengte op. Efficiënter als waarden sterk variëren in lengte.

Gebruik voor: namen, adressen, e-mailadressen.

lastname VARCHAR(50)
-- 'Jansen' (6)
-- 'Van den Berg' (12)
licenseplate VARCHAR(10)
-- '6-VH-RT'
-- kan uitbreiden
📌 TEXT — Grote teksten

Voor lange, onbegrensde teksten. Geen maximale lengte instellen nodig. Niet geschikt voor zoeken of sorteren.

Gebruik voor: beschrijvingen, berichten, opmerkingen.

description TEXT
-- 'Dit is een uitgebreide productbeschrijving...'
TypeLengteOpslagruimteBeste voor
CHAR(n) Vast — altijd n Altijd n bytes Codes, postcodes, landcodes
VARCHAR(n) Variabel — max n Alleen werkelijke lengte Namen, adressen, e-mails
TEXT Onbeperkt Dynamisch Beschrijvingen, berichten

Getallen & Datum/Tijd
INT, DECIMAL, DATE, TIME en DATETIME
▼

Numerieke datatypes

🔢
INT

Hele getallen — geen decimalen. Gebruik voor ID's, aantallen, leeftijden.

id INT -- 1, 42, 10000
amount INT -- 6 personen
SIGNED vs UNSIGNED
SIGNED: getal mag negatief zijn (bijv. temperatuur: -10)
UNSIGNED: alleen positief (bijv. leeftijd, ID — nooit negatief)
💰
DECIMAL(m, d)

Decimale getallen met exacte precisie. m = totaal aantal cijfers, d = cijfers na de komma. Gebruik voor bedragen en maten.

salary DECIMAL(10,2)
-- max 10 cijfers, 2 na komma
-- bijv. 2500.50 of 12345678.90

Datum en tijd datatypes

DATE

Alleen een datum. Geen tijd.

YYYY-MM-DD
bijv. 1990-05-23
geboortedatum, besteldate
TIME

Alleen een tijdstip. Geen datum.

HH:MM:SS
bijv. 14:30:00
begintime, endtime
DATETIME

Datum én tijd samen.

YYYY-MM-DD HH:MM:SS
bijv. 2023-01-18 10:45:32
createdate, registrationdate

Datatypes in tools
Hoe zien datatypes eruit in Moqups, Draw.io en anderen?
▼

Elke tool toont datatypes op een iets andere manier, maar de datatypes zelf zijn overal hetzelfde. Hieronder zie je hoe populaire tools ermee omgaan.

🖼️
Moqups ✅ Wij gebruiken dit

In Moqups voeg je datatypes direct toe aan attributen. Voorbeeld: entiteit reservation:

reservation
  • 🔑 id INT
  • ◆ name VARCHAR(100)
  • ◆ date DATE
  • ◆ starttime TIME
  • ◇ amount INT
🎨 Draw.io

Draw.io toont attributen met hun datatype in tekstvorm. Voorbeeld: entiteit event:

event
  • 🔑 id INT
  • ◆ name VARCHAR(50)
  • ◆ date DATE
  • ◆ begintime TIME
  • ◇ endtime TIME
📊 Lucidchart

Lucidchart laat datatypes zien naast de attribuutnaam. Voorbeeld: entiteit product:

product
  • 🔑 id INT
  • ◆ productcode CHAR(5)
  • ◆ description TEXT
  • ◆ createdate DATETIME
💡 Conclusie — tools, zelfde datatypes
Of je nu Moqups, Draw.io, Lucidchart of MySQL Workbench gebruikt — de datatypes zijn overal hetzelfde. Alleen de weergave verschilt. Ken je de datatypes, dan kun je in elke tool werken. ✅

Account aanmaken en eerste ERD tekenen
▼
🖼️ Waarom Moqups?
Moqups is de tool die wij gebruiken om ERD's te tekenen. Het werkt volledig in de browser — geen installatie nodig. Je kunt samenwerken in realtime en ERD's eenvoudig aanmaken met datatypes.

Stap-voor-stap: account & eerste ERD

📥 Stap 1 — Account aanmaken
Ga naar moqups.com en maak een gratis account aan via je schoole-mailadres.

  1. Klik op Sign Up
  2. Kies Sign up with Google en gebruik je schoolaccount
  3. Je komt direct in je persoonlijke dashboard
📂 Stap 2 — Nieuw project aanmaken
In het dashboard maak je een nieuw project aan:

  1. Klik op + New Project
  2. Geef je project een naam, bijv. ERD Module 2
  3. Kies als template Blank — je begint met een leeg canvas
🗂️ Stap 3 — Een entiteit toevoegen
In het canvas voeg je een entiteit toe als tabel:

  1. Klik links in de toolbar op Shapes (of gebruik de zoekbalk)
  2. Zoek op ERD Table of Entity
  3. Sleep de shape naar het canvas
  4. Dubbelklik op de tabel om de naam te wijzigen
💡 Naamgeving
Gebruik altijd kleine letters en enkelvoud voor je entiteitnaam — bijv. employee, niet Employees.
⚙️ Stap 4 — Attributen & datatypes toevoegen
Klik op de tabel om rijen (attributen) toe te voegen:

  • Klik op + Add row om een attribuut toe te voegen
  • Typ de naam van het attribuut, bijv. firstname
  • Voeg het datatype toe achter de naam, bijv. VARCHAR(50)
  • Markeer de primaire sleutel met een 🔑 of PK label
// Voorbeeld: entiteit reservation in Moqups
reservation
  • 🔑 id PK INT
  • ◆ name VARCHAR(100)
  • ◆ date DATE
  • ◆ starttime TIME
  • ◇ amount INT
🔗 Stap 5 — Relaties tekenen
Verbind twee entiteiten met een relatiepijl:

  1. Hover over de rand van een entiteit — je ziet een verbindingspunt verschijnen
  2. Klik en sleep naar de andere entiteit
  3. Kies het type relatie (1-op-1, 1-op-veel, etc.) in de toolbar
  4. Voeg een vreemde sleutel toe aan de juiste entiteit
💾 Opslaan
Moqups slaat automatisch op in de cloud. Je hoeft niet handmatig op Opslaan te klikken. Deel je project via Share rechtsboven om samen te werken.
